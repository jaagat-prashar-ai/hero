3. New report script: pref_pairs/action_space_variance.py

Mirrors maneuver_report.py's structure (pure functions + thin CLI).

- load_rollouts(results_dir): read all pref_pairs_rollouts*.jsonl shards
under a results dir into one DataFrame (rank-sharded output from step 2).
- per_clip_variance(df): group by scene_id (== one clip's fixed scene
here). Per group, with n_rollouts rows:
  - Per-waypoint mean/std/min/max across rollouts for native_accel_mps2
and native_curvature_per_m (stored as JSON lists; verified equal
length across rollouts of the same scene before stacking).
  - Cross-rollout mean/std/min-max range for each scalar feature already
in the row: final_lateral_offset_m, total_heading_change_deg,
mean_acceleration_mps2, mean_deceleration_mps2,
max_deceleration_mps2, initial/final/min_speed_mps.
  - Carries clip_id, scene_id, event_cluster, n_rollouts (flagged if
!= 100, e.g. a preempted/incomplete scene).
- per_cluster_range(per_clip_df): group the above by event_cluster,
report n_clips, and the median/p90/max across clips of each
variance metric (std and range) — this pooled distribution is the
"final range" deliverable: e.g. epsilon_accel_mps2 = p90 of per-clip native_accel_mps2 std, reported alongside the raw median/max so the
choice of percentile is inspectable, not hidden.
- write_report(...): writes
  - pref_pairs/results/action_space_variance_report.json — structured,
machine-readable: {"per_clip": [...], "per_cluster": [...]}, one entry
per clip with its exact clip_id and its metrics as requested.
  - pref_pairs/results/action_space_variance_report.md — human-readable:
one summary table per cluster (n_clips, median/p90 accel & curvature
std, lateral-offset & heading-change std) up top, then a per-clip
appendix table (clip_id, event_cluster, n_rollouts, key stats).
- CLI: python -m pref_pairs.action_space_variance --results_dir <outdir> --out_dir pref_pairs/results.

# judge_reasoning_pairs.py -- LLM-judge audit of reasoning_matched_pairs.jsonl

Added judge_reasoning_pairs.py: an independent claude-fable-5 judge over
build_reasoning_matched_pairs.py's 717 pairs. Shows both traces BLIND
(randomized A/B, never told which is real) against the scene's actual
64-waypoint trajectory (via trajectory_features.extract_features, not the
4-field scalar summary), scores each 0-10 on whether its claim is consistent
with that trajectory, and writes a scalar margin (chosen_score -
rejected_score) plus a QC signal (does the judge's verdict agree with the
construction-time chosen/rejected label; does it detect the right
perturbation_type).

IMPORTANT PROVENANCE NOTE on pref_pairs/results/judged_pairs/judged_pairs.jsonl:
this 717-row file was NOT produced by running judge_reasoning_pairs.py as a
Lilypad workload. It came from an out-of-band run (717 parallel claude-fable-5
calls orchestrated directly, not through `lilypad workload submit`) done to
validate the judge design at full scale before wiring it into this project's
normal pipeline. Real API calls, real judgments, 717/717 succeeded with 0
errors -- but there is no workload_id, it won't show up in `lilypad workload
list/info/logs`, and it wasn't written via judge_reasoning_pairs.py's own
--out_path flag. Treat this file as a one-off snapshot to build on, not as
reproducible provenance -- a real Lilypad run of judge_reasoning_pairs.py
should replace it before this feeds anything downstream (RL training, the
leakage audit) that cares about that.

Results at a glance: 594 agree with construction / 108 disagree / 15 tie;
mean margin +4.44; corruption_type_detected matches perturbation_type on
576/717 (80%). Breaking down disagreement by type -- attribute_swap is by
far the hardest for the judge to catch from trajectory alone (22/119
disagree, only 78/119 type-matched, avg margin 2.7, all well below every
other type): swapping "lead vehicle" for "parked vehicle" doesn't always
change the predicted trajectory shape as clearly as a negation_flip or
causal_flip does, so a kinematically-similar evasive maneuver can look
equally consistent with either story. Worth a closer look before trusting
attribute_swap pairs for RL without this judge's disagreement flag applied.
