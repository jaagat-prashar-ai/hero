The issue was the Lilypad worker nodes don't have ~/.aws/credentials, so the hardcoded oci.oci AWS profile doesn't exist on them, causing boto3 to fail 
when trying to write shards to S3. 

Fix: 
1. build_webdataset.py - only fall back to the oci.oci profile when AWS_ACCESS_KEY_ID isn't already set; pass AWS_ENDPOINT_URL_S3 explicitly to all boto3.client("s3") calls 
2. build_wds_cluster.yaml - added the OCI Chicago endpoint uRL as a constant_environmental_variable and declared AWS_ACCESS_KEY_ID/AWS_SECRET_ACCESS_KEY as required_environmental_variables so Lilypad passes them from the launch machine to the workers
3. launch_build_wds.sh - reads crdentials from the local oci.oci profile via aws configure get and exports them before launching, so Lilypad picks them up 


Another fix: code actually ran and tried to upload to S3.
